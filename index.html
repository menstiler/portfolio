Some technologies I use

I was first introduced to coding while working as a product specialist on website offering online services, as product specialist it was my responsibility to keep products up to date and develop more modern web and email templates when required. I used HTML and CSS to build responsive website and email templates.

The real coding journey began with the Flatiron School, where I studied Ruby and JavaScript and built apps using frameworks like Ruby on Rails and React. When first introduced to ruby, I was content, it made sense both in its syntax and semantics, it was also built with developers in mind, a motto of the ruby community is: "Matz is nice and so we are nice" referring to the founder of the language, Yukihiro Matsumoto. Then jumping into the wild west of JavaScript there was no looking back. Manipulating the DOM felt like magic compared to the request-response cycle the MVC structure of Rails relies on.

React brought with it everthing that was lacking when I first started writing JavaScript - Structure, Pattern, Convention. It's a beautiful library with an emphasis on building painless user interfaces, it is declartive, component based and an overall pleasure to work with.
